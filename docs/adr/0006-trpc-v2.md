# ADR 0006 — tRPC v11 (vs REST) V2

## Status

Aceita.

## Contexto

V1 usa REST Fastify + Zod schemas declarados duas vezes (backend + cópia no frontend via duplicação manual). Drift entre back e front foi detectado em 3+ endpoints durante o scan.

## Decisão

V2 expõe API via **tRPC v11**. Routers em `apps/api/src/features/*/router.ts`, type `AppRouter` exportado de `packages/contracts/`. Frontend importa o type + cria client tipado.

Estrutura:

```ts
// apps/api/src/features/contacts/router.ts
export const contactsRouter = router({
  list: protectedProcedure.input(listFilterSchema).query(...),
  get: protectedProcedure.input(z.object({ id: z.number() })).query(...),
  create: protectedProcedure.input(createContactInputSchema).mutation(...),
  // ...
});

// apps/api/src/index.ts
export const appRouter = router({
  contacts: contactsRouter,
  conversations: conversationsRouter,
  // ...
});
export type AppRouter = typeof appRouter;
```

Frontend:

```ts
// apps/web/src/lib/trpc.ts
import type { AppRouter } from "@nuoma/contracts";
export const trpc = createTRPCReact<AppRouter>();

// apps/web/src/features/contacts/list.tsx
const { data } = trpc.contacts.list.useQuery({ cursor: null, limit: 50 });
//          ^? Contact[] inferido automaticamente
```

Streaming + REST quando precisar:

- `tRPC` cobre 95% das interações CRUD.
- **REST** mantido em `/api/streaming/ws` (WebSocket relay CDP), `/api/embed/quick-send` (legacy/extension futura), `/api/uploads/media` (multipart).
- **SSE** em `/api/events/stream` (Hono nativo, fora do tRPC).

## Consequências

- **Bom**: Type safety end-to-end. Sem drift back/front. IntelliSense nas chamadas. Refactor de procedure quebra typecheck do frontend imediatamente.
- **Custo**: Curva pra quem não conhece tRPC. Mas é semelhante a React Query + Zod.
- **Bind**: tRPC tem boa portabilidade — pode abandonar e ir pra REST depois sem grande dor (basta inverter os schemas).

## Alternativas

- REST + OpenAPI codegen: viável mas burocrático (yaml, gen step).
- GraphQL: descartado — overkill pra single-tenant single-user.
- ts-rest: viável mas tRPC tem ecossistema mais maduro.
